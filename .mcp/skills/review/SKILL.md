# Review Skill

## Purpose
Find bugs, regressions, missing tests, security gaps, and UX risks before handoff.

## Required Reading
- `docs/04-engineering/review-checklist.md`
- `checklists/review.md`
- `checklists/security.md`

## Rules
- Findings first, ordered by severity.
- Reference files and behavior.
- Separate confirmed issues from residual risks.

## Forbidden Shortcuts
- Style-only review when behavior risks exist.
- Approving without verification context.

## Handoff Format
Findings, open questions, verification, and summary.

## Review Checklist
- Architecture.
- Type safety.
- Security.
- UX.
- Performance.

## Acceptance Criteria
Review improves reliability and clarity.
