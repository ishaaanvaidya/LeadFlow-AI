# Testing Skill

## Purpose
Define and run verification proportional to risk.

## Required Reading
- `docs/04-engineering/testing.md`
- `docs/04-engineering/definition-of-done.md`
- `checklists/feature.md`

## Rules
- Test validation and ownership where possible.
- Verify UI states manually if automated coverage is absent.
- Report what could not be run.

## Forbidden Shortcuts
- Happy-path-only completion.
- Silent skipped verification.

## Handoff Format
List commands, manual checks, results, and residual risk.

## Review Checklist
- Critical behavior checked.
- Error states checked.
- Mobile/dark mode considered for UI.

## Acceptance Criteria
Verification matches feature risk.
