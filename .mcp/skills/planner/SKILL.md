# Planner Skill

## Purpose
Turn user requests into ordered, documentation-aware implementation steps.

## Required Reading
- `.context/project-summary.md`
- `.context/current-phase.md`
- `.context/next-task.md`
- `docs/04-engineering/feature-lifecycle.md`

## Rules
- Identify scope, dependencies, and verification.
- Read docs before proposing implementation.
- Use templates and checklists.

## Forbidden Shortcuts
- Starting with code before behavior is clear.
- Ignoring current phase.

## Handoff Format
Provide goal, files likely touched, order of work, tests, and risks.

## Review Checklist
- Scope matches PRD.
- No post-MVP drift.
- Verification named.

## Acceptance Criteria
An implementer can start without asking what to build.
