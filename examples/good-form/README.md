# Example: Good Form

## Example Purpose
Show form expectations for CRM input.

## Pattern
A good form has labels, validation messages, disabled pending state, submit-level error, cancel path, and clear success behavior.

## Good Qualities
- Accessible fields.
- Server validation.
- Friendly errors.
- No data loss on recoverable failure.
