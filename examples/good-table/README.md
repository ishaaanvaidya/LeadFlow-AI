# Example: Good Table

## Example Purpose
Show CRM table expectations.

## Pattern
A good leads table supports visible columns, sorting, filtering, keyboard navigation, empty state, loading state, and pagination.

## Good Qualities
- Dense but readable.
- Actions are discoverable.
- Rows link to detail pages.
- Status and score are scannable.
