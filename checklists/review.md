# Review Checklist

## Checklist
- Behavior matches request.
- Architecture matches docs.
- Types are strict.
- Validation exists.
- Security boundaries hold.
- Error states are safe.
- UI is accessible and responsive.
- Tests or manual verification match risk.
- No undocumented pattern introduced.
